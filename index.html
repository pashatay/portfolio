Dream Log helps people store and share their dreams. Dreams can be categorized as lucid, normal or nightmare and the UI visually supports it. A user provides a description and category for each dream, along with hours of total sleep. The provided data is used by the application to build personal statistics for each user. Users can decide which dreams they would like to share publicly in a the built in dream blog. Shared dream blogs can be accessed by anyone including unregistered users.
Dream Log also includes email verification, change of user credentials, and password reset capabilities.

Techonologies I used to build this: React (Router, Context, Hooks), Node/Express, JWT, Bcrypt, XSS, NodeMailer, HBS, Chart.js, PostgreSQL, CSS.
